# Offline Audit

## ما الحقيقي؟

1. construction documentation workflows مؤلمة جدًا.
2. المواصفات والرسومات والعقود ضخمة ومعقدة.
3. RFIs/submittals/change orders قابلة للأتمتة الجزئية.
4. AI يجب أن يعطي citations إلى specs/drawings.
5. human review إلزامي بسبب liability.
6. BIM/vision مهم، لكن الوثائق أسهل كـ MVP.
7. المجال أقل تشبعًا من legal/healthcare في document AI المتخصص.

## ما الـ hype؟

1. AI project manager كامل.
2. BIM + AI يحل كل المخاطر.
3. AI يرد على RFIs دون مراجعة مهندس.
4. document Q&A بلا citation.
5. schedule risk prediction بلا بيانات جيدة.

# التحليل العميق

## 1. construction هو domain مثالي لـ evidence-grounded document AI

كل إجابة يجب أن تستند إلى:

- spec section
- drawing reference
- addendum
- contract clause
- code requirement
- RFI history

هذا مثل legal/finance، لكن ببيانات project-specific.

## 2. Submittals/RFIs هي أفضل wedge

لأنها:

- متكررة.
- مكلفة زمنيًا.
- قائمة على specs.
- قابلة للـ source citation.
- تقبل human review.

## 3. construction knowledge يتغير بالمشروع

كل مشروع له:

- specs
- drawings
- addenda
- RFIs
- submittals
- change orders

إذن النظام يجب أن يبني project-specific knowledge base.

## 4. closeout docs فرصة كبيرة

في نهاية المشروع، تجميع:

- warranties
- O&M manuals
- as-builts
- submittals
- test reports

مؤلم جدًا ومناسب لـ document automation.

# Candidate Theses

## Thesis A — Submittal Log / Requirement Extractor

### المشكلة

قراءة مواصفات طويلة لاستخراج submittal requirements تستغرق ساعات/أيام.

### الأطروحة

> أداة تستخرج submittal requirements من specs وتولد log مع references.

### MVP

- upload spec PDF
- extract requirements
- section/page citation
- export Excel

### قوي جدًا لأنه

واضح، مؤلم، ومثبت في case studies.

---

## Thesis B — RFI Drafting & Evidence Assistant

### المشكلة

RFIs تحتاج صياغة دقيقة مع references.

### الأطروحة

> يكتشف التناقض/الغموض في specs/drawings ويقترح RFI draft مع evidence.

### MVP

- user question + docs
- retrieve relevant sections
- draft RFI
- cite sources

---

## Thesis C — Spec Compliance Checker

### المشكلة

Submittals/materials يجب أن تطابق specs.

### الأطروحة

> يقارن product submittal ضد spec requirements ويخرج compliance table.

### MVP

- spec section + product data sheet
- extract required properties
- compare submitted properties
- flag missing/noncompliant

### قوي جدًا وقابل للقياس.

---

## Thesis D — Change Order Evidence Pack Builder

### المشكلة

Change orders تحتاج ربط السبب بالعقد/RFI/schedule/cost.

### الأطروحة

> يبني evidence pack لدعم/مراجعة change order.

### MVP

- change request + docs
- evidence links
- risk/validity checklist

---

## Thesis E — Construction Closeout Document Assembler

### المشكلة

closeout مؤلم ومليء بالمستندات.

### الأطروحة

> يجمع ويصنف closeout docs ويكشف missing warranties/O&M/as-builts.

### MVP

- folder of docs
- classify
- checklist
- missing docs report

---

## Thesis F — Project Document Risk Auditor

### المشكلة

المشاريع تخفي مخاطر في RFIs/submittals/specs/change orders.

### الأطروحة

> يحلل project docs ويخرج risk register مع evidence.

### أوسع وأصعب.
